# Pace — nr-next-marketing

A demo marketing website built as a **day-by-day interview study project** for a
Senior Front-End role (React / Next.js / Tailwind / Contentful). Stack: **Next.js 16**
(App Router, Turbopack, Cache Components/PPR), **React 19** (React Compiler on),
**TypeScript**, **Tailwind v4**, **shadcn/ui + Radix**. Brand name: **Pace**.

Each "day" teaches a topic by building **real, runnable** features into this one app,
then writing a study doc. Days 1–15 are done; **Days 16–18 remain** (see the study
plan in `README.md`).

## Commands
```bash
npm run dev        # local dev (http://localhost:3000)
npm run build      # production build (also type-checks + prerenders)
npm run lint       # eslint (incl. full jsx-a11y recommended)
npm run typecheck  # next typegen && tsc --noEmit
npm test           # vitest (unit tests for pure utils)
```

## How to continue the study (the per-day method — follow this)
When asked to "do Day N" (or continue), replicate the established loop:
1. **Pick** the next unchecked day in the study plan in `README.md` (it lists topic + build task + interview Qs).
2. **Read the Next 16 docs first.** Per `AGENTS.md`, this Next differs from training
   data — read the relevant guide under `node_modules/next/dist/docs/` before writing
   any Next.js code, and heed deprecations.
3. **Build real, runnable code** into the app (extend existing pages/components where
   possible; don't write throwaway snippets). Keep comments matching the surrounding
   style (they explain the "why" for study).
4. **Verify with proven output.** There is **no headless browser** here, so verify via:
   `npm run build` (route table), `npm run start` + `curl`/`grep` of served HTML, and
   computed checks (e.g. contrast ratios, parsing JSON-LD). Be **honest** about what
   couldn't be run (Lighthouse/axe/real Contentful/Marketo/GTM use fixtures/fallbacks).
5. **Write the study doc** `docs/dayNN_<topic>.md` in the established format (below).
6. **Tick the day** in the study plan in `README.md`.
7. **Offer a compact commit message** (don't commit/push unless asked; `main` push
   auto-deploys to Vercel).

### Study-doc format (match the existing day docs)
`# Day N — Title` → blockquote target/intro → **Recap** table → **Abbreviations**

## Conventions & Next 16 gotchas (learned across Days 1–13)
- **Renames/changes vs older Next:** `middleware.ts` → **`proxy.ts`**; `revalidateTag(tag, "max")`
  (two-arg); `draftMode()`/`cookies()`/`params`/`searchParams` are **async**; route types via
  `PageProps<"/route">` / `RouteContext<"/route">` (generated by `next typegen`).
- **Cache Components is ON** (`cacheComponents: true`): caching is **opt-in** —
  `use cache` + `cacheLife` + `cacheTag`; runtime APIs (cookies/searchParams) must live in
  a `<Suspense>` boundary (PPR). `draftMode().isEnabled` **is** readable inside `use cache`.
  On-demand revalidate via `POST /api/revalidate?secret=…&tag=…`.
- **React Compiler is ON** (`reactCompiler: true`): don't hand-write `useMemo`/`useCallback`/`memo`.
  For DOM/external state use **`useSyncExternalStore`**, not effect-then-setState.
- **Tailwind v4 (CSS-first):** tokens in `src/app/globals.css` `@theme` (brand-* in OKLCH;
  semantic `surface`/`ink`; shadcn tokens). Dark mode = **`:root.dark`** block (the extra
  specificity matters — a plain `.dark` loses to shadcn's later `:root`). `cn()` from
  `@/lib/utils` (re-exports `@/lib/cn` = clsx + tailwind-merge).
- **Accessibility:** eslint runs **jsx-a11y recommended**. Verify contrast in **both** light
  and dark (use theme-aware color pairs, e.g. `text-brand-600 dark:text-brand-300`); control
  borders ≥ 3:1; keep the skip link + visible `:focus-visible`.
- **Untrusted data** (CMS/API) is validated with **zod** at the boundary (`src/lib/cms`,
  `validate()`); `server-only` modules can't be imported in client code or tests.
- **shadcn/ui** lives in `src/components/ui` (copy-in source you own); Radix unified pkg.

## Where things live
- **App:** `src/app/(marketing)/*` (pages), `src/app/api/*` (route handlers),
  `src/proxy.ts` (edge A/B), `src/components` (+ `ui/` = shadcn), `src/lib` (+ `cms/`).
- **Study:** the study plan lives in `README.md` (schedule + tracker + day index);
  `docs/` holds `dayNN_*.md` (notes) and `deployment_vercel.md` (deploy guide).
- **Tests:** `src/**/*.test.ts` (vitest; pure utils only — avoid `server-only` imports).
- **CI:** `.github/workflows/ci.yml` (gates); `deploy.yml` is manual-only.

## Deploy
Live on **Vercel** via native Git integration — every push to `main` auto-deploys prod,
branches/PRs get preview URLs. Env vars are documented in `.env.example`
(all optional; the app runs on fixtures/fallbacks). Set `NEXT_PUBLIC_SITE_URL` to the
deployed URL for correct canonicals/OG/sitemap. See `docs/deployment_vercel.md`.
